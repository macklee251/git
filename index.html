Curso de Git

Comandos

  • git add *.html coloca no stage totos os arquivos .html
  • git restore remove o arquivo do stage (Volta para a versão do diretório)
  • podemos colocar todos os arquivos no stage e dar commit ao mesmo tempo com o comando git commit -am "Mensagem"
  • podemos ajustar o ultimo commmit que foi feito com o comando git commit --amend -m "Mensagem"
  • podemos ver um gráfico das logs como: git log --graph
  • podemos ver uma versão simplificada dos logs com git shortlog
  • usamos o git remote -v para ver a url do remote
  • podemos usar o git remote remove origin para remover o remote origin
  • Para trazer todas as informações do repositório remoto para o local usamos o comando git pull origin main
  • O comando git remote show origin mostra informações sobre o remote

é uma boa prática fazer um git pull antes de começar a trabalhar em um projeto e fazer o git pull antes de dar pull, assim garantimos que estamos atualizados com as ultimas mudanças do repositório.


Branches

  1. main
  2. release
  3. develop
  4. feature
  5. fix

Podemos criar um branch chamada dev e fazer todo o desenvolvimento por ali

  • podemos criar um branch com git branch dev ou git checkout -b dev
  • para renomear podemos dar o comando git branch -m dev develop
  • podemos excluir um branch com git branch -D dev
  • para ver o log de um branch em espefífico podemos fazer git log main por exemplo
